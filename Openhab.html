Openhab

In deze activiteit heb ik een Raspberry Pi opgezet die fungeert als “home controller”. We hebben al deels automatisering in huis met niko home control (NHC). Er zijn aan dit systeem echter bepaalde zaken die mij storen. De interface is niet al te gebruiksvriendelijk en het grootste nadeel is dat de connectiviteit met andere spelers zeer beperkt is. Niko werkt samen met zijn partners, maar als je nu net een device van een andere fabrikant wilt integreren, dan lukt dat niet. Via Openhab kan je via code of via de UI apparaten laten samenkomen op 1 systeem..

We starten bij de Raspberry Pi. Deze fungeert als server waarop alle signalen gaan binnenkomen. Speciaal hiervoor bestaat er Openhabian. Je kan het flashen op een micro-SD kaart en vervolgens in de Raspberry Pi steken. Wanneer je deze aansluit begint de server zich automatisch te installeren. Na een goed half uur is hij bereikbaar via de localhost nl. openhabian:8080. Je kan hier je wachtwoord instellen en vanaf dan kan je ook aan de netwerkmappen. Ten slotte kan je ook in de configuratie zelf duiken door connectie te maken via SSH.

Ik ben de activiteit gestart met de 2.5 versie van Openhab. In de loop van het project zijn hier enkele updates aan te pas gekomen en uiteindelijk hebben ze de 3.0 versie gereleased. Dit zorgde voor heel wat problemen in bestaande systemen zoals bij mij. Enkele bindings werkten niet meer omdat er problemen ontstaan waren met de beveiliging.

In Openhab kan je kiezen om alles via een UI in te stellen of anderzijds via de files zelf. In Opehab 2.5 heb ik alles via Visual Studio Code opgezet, maar vermits de 3.0 versie toch verschillend in elkaar zit, heb ik daar via de UI alles ingesteld. Hier is het wel mogelijk om in de UI zelf ook de code aan te passen.

Uiteindelijk heb ik een binding opgezet met mijn Niko Home Control installatie en alles toegewezen aan de locatie in huis. Verder heb ik ook mijn 2 Bose Soundtouch systemen gekoppeld aan mijn keuken en badkamer. Alles is zo gemakkelijk toegankelijk vanuit de mobiele Openhab applicatie. In de vorige versie had ik mijn sproeiersysteem gekoppeld, maar met de upgrade is deze binding niet meer functioneel. Verschillende developers zijn daarmee bezig om deze binding terug werkende te krijgen en beschikbaar te stellen aan iedereen. Vaak wordt er beroep gedaan op API’s van de betreffende apparaten, maar als de fabrikant soms aanpassingen doet, moet de binding in Openhab dus ook bijgewerkt worden.

De grote sterkte van Openhab is dat je ook regels kan schrijven. Zo heb ik bijvoorbeeld het IP-adres van mijn gsm gekoppeld. Wanneer na 21:00 uur de status van dat IP-adres van Off naar On gaat, dan gaat de lamp aan de achterdeur aan. Omdat ik wel eens vergeetachtig ben en de lamp niet altijd uitzet, heb ik ervoor gezorgd dat de lamp in dit geval na 5 minuten weer uit gaat. Nog een ander voorbeeld is dat ik de buitenverlichting aan de inkom een kwartier voor zonsondergang laat aanspringen, om dan weer te doven om 23:30 uur.

Ik kan Openhab zeker aanraden, al is het wel een must kennis te hebben van servers/programmeertaal/API’s,.. . Ik ga het ook zeker verder blijven gebruiken om nog andere apparaten te koppelen.
